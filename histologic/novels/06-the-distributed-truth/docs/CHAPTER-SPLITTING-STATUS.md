## üìù NOTES ON SPLITTING PROCESS

### Pattern for Splitting:
1. Read combined file
2. Identify chapter boundaries (marked by `## Chapter N:` headers)
3. Extract each chapter with:
   - Header: `# Chapter N: [Title]`
   - POV line: `## POV: [Character Name]`
   - Timeline/Setting
   - Full chapter content
   - Footer: `**End of Chapter N**` + word count
4. Create individual files
5. Delete combined file
6. Update TODO list

### File Naming Convention:
- Format: `NN-chapter-title-kebab-case.md`
- Examples:
  - `12-the-first-crisis.md`
  - `13-the-coordinated-attack.md`
  - `14-old-songs-for-new-times.md`

### Why Some Remain Combined:
- Chapter summaries are complete (more important for reference)
- Combined files are functional and readable
- Splitting is organizational improvement, not critical
- Can be done anytime without affecting novel completeness
